    elasticsearch cluster-reroute

    Change the allocation of shards in a cluster.

    Description

    Change the allocation of shards in a cluster.

    Synopsis

    elasticsearch cluster-reroute
    [--site <site>]
    [--cluster <cluster>]
    [--dry_run]
    [--explain]
    [--retry_failed]
    --command <command>
    --index <index>
    --shard <shard>
    [--node <node>]
    [--to_node <to_node>]
    [--allow_primary]

    Arguments

    site - (string)

         Site where this command will be executed
         Example: --site "site-1"
         Default: input.site      Attributes: optional

    cluster - (string)

         Name of elastic search cluster
         Example: --cluster "elastic-default"
         Default: elastic-default      Attributes: optional

    dry_run - (bool)

         If set, then the request simulates the operation and returns the resulting state, but will not actually perform the requested changes.
         Example: --dry_run
         Default: _None_      Attributes: optional

    explain - (bool)

         If set, then the response contains an explanation of why the commands can or cannot be executed.
         Example: --explain
         Default: _None_      Attributes: optional

    retry_failed - (bool)

         If set, then retries allocation of shards that are blocked due to too many subsequent allocation failures
         Example: --retry_failed
         Default: _None_      Attributes: optional

    command - (string)

         Command to perform. Can be either move/ cancel/ allocate_replica
         Example: --command "move"
         Default: _None_      Attributes: required

    index - (string)

         Index on which command is performed
         Example: --index "students"
         Default: _None_      Attributes: required

    shard - (int)

         Shard on which command is performed
         Example: --shard 1
         Default: _None_      Attributes: required

    node - (string)

         Node from which shard should move or node from which shared allocation should be cancelled or node to which unassigned replica should be allocated
         Example: --node "node2"
         Default: _None_      Attributes: optional

    to_node - (string)

         In case of move command, node to which shard is moved
         Example: --to_node "node2"
         Default: _None_      Attributes: optional

    allow_primary - (bool)

         By default only replica shard allocations can be cancelled. If it is necessary to cancel the allocation of a primary shard then the allow_primary flag must be set
         Example: --allow_primary
         Default: _None_      Attributes: optional

    Access Control

    To use this command, you need access to the following:

    Field Value
    Action "update"
    Service Type "elasticsearch"
    Service Instance cluster
    Namespace
    Object Type "index"
    Object ID index
